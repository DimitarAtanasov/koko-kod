# Коко Код Академия

Приказна платформа за учене на програмиране — Курс 6+ (Роби) и Курс 11+ (Луна, Бо и Мия).

## Файлове
- `index.html` — HTML шаблонът (нав, онбординг, настройки) и връзки към стиловете/скриптовете по-долу
- `styles.css` — всички стилове
- `js/core.js` — съхранение на прогреса, озвучаване, интереси, онбординг/настройки, стартиране на приложението
- `js/track-a.js` — Курс 6+ (Роби, лабиринти с блокчета)
- `js/track-b.js` — Курс 11+ (Луна, Бо и Мия — истински C++ основи)
- `manifest.json` — PWA манифест (име, икони, цветове)
- `service-worker.js` — офлайн кеширане
- `icon-192.png`, `icon-512.png` — икони за инсталация

Всички файлове + `.nojekyll` трябва да са в **корена на репото** (папката `js/` е ок, само относителните пътища в `index.html`/`service-worker.js` трябва да съвпадат). Няма стъпка на компилиране — файловете се сервират директно както са, GitHub Pages просто ги вдига статично.

## Публикуване през GitHub Pages
1. Settings → Pages → Source: "Deploy from a branch" → Branch: `main` / `(root)` → Save
2. След ~1 минута сайтът е на `https://<потребител>.github.io/<репо>/`

## Обновяване по-късно
При всяка промяна на кой да е от файловете в app shell-а (`index.html`, `styles.css`, `js/*.js`, `manifest.json` или иконите) — **повиши номера** в `service-worker.js`:
```js
const CACHE_NAME = 'koko-kod-v3';  // v1 -> v2 -> v3 ...
```
Иначе устройства, които вече са инсталирали приложението, ще продължат да виждат старата кеширана версия.

## Прогрес на децата
Пази се локално на устройството през `localStorage` (с резервен вариант през вградената памет на артефакта `window.storage`, ако приложението работи в такава среда). Не се синхронизира между устройства — това е следваща стъпка, ако платформата порасне (истински бекенд с акаунти).
